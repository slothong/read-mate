# Doc Explainer - PRD

## 1. 개요

웹 페이지에서 텍스트를 드래그하여 선택하면, AI가 해당 내용을 쉽게 설명해주는 Chrome 익스텐션.

## 2. 문제 정의

- 웹 문서를 읽다가 이해하기 어려운 용어, 문장, 코드 등을 만났을 때 별도로 검색하거나 탭을 열어야 하는 번거로움
- 전문 용어, 외국어, 기술 문서 등 즉석에서 맥락에 맞는 설명이 필요한 상황이 빈번함

## 3. 목표

- 텍스트 선택만으로 2초 이내에 AI 설명을 표시
- 페이지 이탈 없이 인라인으로 결과 확인
- 다양한 종류의 콘텐츠(일반 텍스트, 코드, 수식 등)에 대응

## 4. 사용자 시나리오

### 시나리오 1: 전문 용어 설명
1. 사용자가 기술 문서에서 "Raft consensus algorithm" 텍스트를 드래그
2. 선택 영역 근처에 툴팁 버튼("설명해줘")이 표시됨
3. 버튼 클릭 시 AI가 해당 용어를 쉬운 말로 설명하는 팝오버가 나타남

### 시나리오 2: 코드 스니펫 설명
1. 사용자가 블로그의 코드 블록 일부를 선택
2. 툴팁 버튼 클릭
3. AI가 코드의 동작을 단계별로 설명

### 시나리오 3: 긴 문단 요약
1. 사용자가 긴 문단을 드래그하여 선택
2. 툴팁 버튼 클릭
3. AI가 핵심 내용을 요약하여 표시

## 5. 기능 요구사항

### 5.1 텍스트 선택 감지
- 사용자가 페이지에서 텍스트를 드래그하여 선택 완료(mouseup) 시 감지
- 선택된 텍스트가 최소 2자 이상일 때만 동작
- 선택된 텍스트가 10,000자를 초과하면 잘라서 처리하고 사용자에게 알림

### 5.2 트리거 UI (툴팁 버튼)
- 텍스트 선택 완료 시 선택 영역 우측 상단에 작은 플로팅 버튼 표시
- 버튼 아이콘: 💡 또는 커스텀 아이콘
- 다른 영역 클릭 시 버튼 자동 소멸
- 페이지의 기존 UI와 충돌하지 않도록 z-index 최상위 + Shadow DOM 적용

### 5.3 설명 팝오버
- 트리거 버튼 클릭 시 선택 영역 근처에 팝오버 표시
- 팝오버 구성:
  - 헤더: 선택한 텍스트 미리보기 (말줄임 처리)
  - 본문: AI 생성 설명 (마크다운 렌더링 지원)
  - 푸터: "복사" 버튼, "닫기" 버튼
- AI 응답 대기 중 로딩 스피너 표시
- 스트리밍 응답 지원: 토큰 단위로 점진적 렌더링
- 팝오버 외부 클릭 시 닫힘
- 팝오버는 드래그하여 위치 이동 가능

### 5.4 AI API 연동
- **지원 API**: OpenAI API (GPT-4o 등)
- 요청 구성:
  - System prompt: 선택된 텍스트의 종류(일반, 코드, 수식 등)에 맞춰 설명하도록 지시
  - User message: 선택된 텍스트 + 주변 맥락 (선택 영역 앞뒤 500자)
- 스트리밍 모드(SSE) 사용하여 응답 점진 표시
- 타임아웃: 30초
- 에러 발생 시 팝오버 내에 재시도 버튼 표시

### 5.5 설정 (팝업 페이지)
- API Key 입력 및 저장 (chrome.storage.sync)
- AI 모델 선택 (예: gpt-4o-mini, gpt-4o)
- 설명 언어 설정 (한국어/영어/자동감지)
- 익스텐션 활성화/비활성화 토글

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 성능 | 툴팁 버튼 표시까지 100ms 이내, API 첫 토큰까지 2초 이내 |
| 보안 | API Key는 chrome.storage.sync에 암호화 없이 저장 (V1), 외부 서버 경유 없이 클라이언트에서 직접 API 호출 |
| 호환성 | Chrome 120+ 지원 |
| 접근성 | 키보드 단축키(Ctrl/Cmd+Shift+E)로도 설명 트리거 가능 |
| 충돌 방지 | Shadow DOM으로 페이지 CSS와 격리, Content Security Policy 준수 |

## 7. 기술 아키텍처

```
[Content Script]           [Background Service Worker]       [External]

 텍스트 선택 감지 ──────>  (필요 시 API 호출 중계)
 툴팁 버튼 표시
 버튼 클릭 ─────────────> API 요청 전송 ──────────────> OpenAI API
 팝오버 렌더링 <───────── 스트리밍 응답 중계 <──────────

[Popup]
 API Key 설정
 모델/언어 설정
```

### 컴포넌트 구성

| 파일 | 역할 |
|------|------|
| `manifest.json` | 익스텐션 설정, 권한 선언 |
| `content.js` | 텍스트 선택 감지, 툴팁/팝오버 UI 렌더링 |
| `content.css` | 툴팁/팝오버 스타일 |
| `background.js` | Service Worker. API 호출 중계, 설정 관리 |
| `popup.html/js/css` | 설정 페이지 UI |
| `lib/api.js` | OpenAI API 호출 로직 |

### API 호출을 Background에서 처리하는 이유
- Content Script는 페이지의 CSP(Content Security Policy)에 제약받아 외부 API 호출이 차단될 수 있음
- Background Service Worker는 CSP 제약 없이 자유롭게 외부 요청 가능
- API Key가 Content Script 컨텍스트에 노출되지 않도록 격리

## 8. 데이터 흐름

1. **텍스트 선택** → Content Script가 `mouseup` 이벤트에서 `window.getSelection()` 호출
2. **트리거** → 플로팅 버튼 클릭 시, Content Script가 `chrome.runtime.sendMessage()`로 Background에 요청 전송
3. **API 호출** → Background가 저장된 API Key로 OpenAI API 스트리밍 요청
4. **응답 전달** → Background가 `chrome.runtime.Port`를 통해 Content Script에 청크 단위 전달
5. **렌더링** → Content Script가 팝오버에 점진적으로 텍스트 추가

## 9. 마일스톤

### M1: 기본 동작 (MVP)
- 텍스트 선택 → 툴팁 버튼 → 팝오버에 AI 설명 표시
- 팝업에서 API Key 설정
- OpenAI API 연동 (non-streaming)

### M2: UX 개선
- 스트리밍 응답 지원
- 마크다운 렌더링
- 팝오버 드래그 이동
- 키보드 단축키

### M3: 고도화
- 추가 AI 프로바이더 지원 (Claude 등)
- 설명 히스토리 저장
- 설명 톤/깊이 커스터마이징 (간단히 / 자세히)
- 페이지별 활성화/비활성화

## 10. 범위 외 (Out of Scope)
- 자체 백엔드 서버 구축
- 사용자 계정/인증 시스템
- PDF, 이미지 내 텍스트 인식
- 모바일 브라우저 지원
- Firefox/Safari 등 타 브라우저 지원
